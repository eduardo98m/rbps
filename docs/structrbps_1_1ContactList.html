Detailed Description

Unified SoA carrying the pipeline output AND the solver scratch state.

Replaces the old ContactCollection. Fields are grouped by lifecycle:

  • Pipeline output — filled by run_narrow_phase, read-only during the solve.
  • Solver state — zeroed at frame start, written every substep.
Body indices vs IDs
body_a / body_b store raw BodyCollection slot indices, not user-facing IDs. The pipeline does the conversion at emit time, so the solver can index bc.position[body_a[i]] directly without an extra ID-to-index lookup.
