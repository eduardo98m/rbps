Define a named dynamic SoA struct with direct field access.

Define a named dynamic SoA struct with direct field access.Same API as DEFINE_SOA, but capacity grows at runtime. Fields become std::vector members; access is still bc.position[i].

Warning
Pointers into field vectors are invalidated on reallocation. Call bc.reserve(n) before a burst of bc.add() calls if you need stable pointers (e.g. before constructing a view for a solver loop).
Example
DEFINE_DYN_SOA(ParticleCollection, uint32_t, 8, PARTICLE_FIELDS)
ParticleCollection pc;
uint32_t id = pc.add();
pc.position[pc.index_of(id)] = {0, 5, 0};
for (uint32_t i = 0; i < pc.count(); ++i)
pc.position[i] += pc.velocity[i] * dt;

// --- 7.10 Stress test [cite: 157] ---
TEST(interleaved_add_remove) {
// Add 10 items. Remove every other one. Assert count()=5. Assert 5 surviving
// IDs map correctly. [cite: 158, 159]
SimpleCollection c;
std::vector<uint32_t> ids;
for (int i = 0; i < 10; ++i) {
uint32_t id = c.add();
c.value[c.index_of(id)] = i * 100;
ids.push_back(id);
}
for (int i = 0; i < 10; i += 2) {
c.remove(ids[i]);
}
ASSERT_EQ(c.count(), 5);
for (int i = 1; i < 10; i += 2) {
ASSERT_TRUE(c.contains(ids[i]));
ASSERT_EQ(c.value[c.index_of(ids[i])], i * 100);
}
// Add 5 more items. Assert count() equals 10. [cite: 160]
c.add_n(5, [](uint32_t) {});
ASSERT_EQ(c.count(), 10);
}
