Detailed Description

Macro-generated SoA collections with stable versioned IDs.

A concrete collection is created with DEFINE_SOA (fixed capacity) or DEFINE_DYN_SOA (dynamic capacity). Both produce a struct with one named array per field and an O(1) add / remove / index_of API, with generation-bit versioning to detect stale IDs.

In tight solver loops, callers convert ID → packed data index once and then access fields directly:

for (uint32_t i = 0; i < bc.count(); ++i)
bc.position[i] += bc.linear_velocity[i] * dt;

Define a named SoA struct with direct field access.

Generates a struct where every field from FIELDS_MACRO becomes a plain array member, giving clean bc.position[i] access in loops.

Parameters
NameName of the generated struct (e.g. BodyCollection).
IdTypeInteger type for IDs (e.g. uint32_t).
CapCompile-time max item count.
GenBitsBits reserved for the generation counter (0 = disabled).
FIELDS_MACROX-macro listing fields as X(Type, name).
Minimal example
#define PARTICLE_FIELDS(X) \
X(vec3, position) \
X(vec3, velocity) \
X(float, mass)
DEFINE_SOA(ParticleCollection, uint32_t, 1024, 8, PARTICLE_FIELDS)
ParticleCollection pc;
uint32_t id = pc.add();
uint32_t i = pc.index_of(id);
pc.position[i] = {0, 5, 0};
pc.mass[i] = 1.0f;
// Solver loop — same as the old struct-of-vectors style
for (uint32_t i = 0; i < pc.count(); ++i)
pc.position[i] += pc.velocity[i] * dt;
pc.remove(id);
pc.contains(id); // false
