# RBPS — Rigid Body Physics Simulator

A small, modular C++17 rigid-body physics engine. This site contains the API
reference and the design documents for upcoming features.

## Modules

| Module      | Purpose                                                                          |
| ----------- | -------------------------------------------------------------------------------- |
| **math3d**  | Foundational 3D math — `vec3`, `quat`, `mat3`, transforms (header-only).         |
| **storage** | Structure-of-Arrays containers generated by macro (`DEFINE_DYN_SOA`).            |
| **ivc**     | Index Vector Core (IVC) — stable ID management for Struct-of-Arrays collections. |
| **rbc**     | Rigid-body collision: shapes, broad-phase, narrow-phase, GJK/EPA.                |
| **rbps**    | Physics core: bodies, constraints, joints, contacts, the `World` API.            |
| **visr**    | Optional visualizer built on raylib + ImGui (gated by `RBPS_BUILD_VISR`).        |

The dependency order is `math3d → storage/ivc → rbc → rbps → visr`.

## Building the docs locally

```bash
cmake -DRBPS_BUILD_DOCS=ON -B build
cmake --build build --target docs
xdg-open build/docs/html/index.html
```

## Where to start

- **Using the engine:** open the `rbps` module page and read the `World` class.
- **Adding a new shape:** start at the `rbc/shapes` group.
- **Contributing a new feature:** see the @ref planned_features "Planned Features" section in the sidebar.
- **Writing documentation:** read [STYLE.md](STYLE.md) for the docstring conventions.

## Internals

Algorithm implementations (GJK, EPA, constraint solver, broad-phase data
structures) are documented under the **Internals** group. They are not part
of the public API but are valuable when maintaining or extending the engine.
